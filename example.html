部署文档

本文档将介绍如何搭建 GitHub.io 博客，并部署基于 jemdoc 的学术风格博客

博客项目代码
[参考，知乎 - Jemdoc + Github.io + Mac搭建个人学术主页]

提前准备

  • 安装 Python，用于执行 jemdoc.py 脚本

搭建 GitHub 博客

[参考，CSDN - 搭建自己的github.io博客]

  1. 新建一个命名为 {YOUR_NAME}.github.io 代码库

  2. 进入 "设置（setting）" -> "页面（Pages）" 完成配置

  3. 当页面显示 "Your site is published at https:{YOUR_NAME}.github.io/" 时，便可直接访问该域名。

编辑基于 jemdoc 的学术风格博客页面

  1. 拉取仓库代码到本地，进入该项目目录，准备编辑（原来的初始化仓库时生成的文件都可以删掉）

  2. 从 jemdoc 官网 下载 jemdoc.py 和 jemdoc.css 两个文件，存到项目目录下

  3. 创建空的模板文件 index.jemdoc，按照示例模板或教程的案例编辑内容

  4. 执行命令 # python2 jemdoc.py index.jemdoc，将编译得到最终使用的 html 文件（每次更新了 .jemdoc 模板文件后都要执行一次该命令以刷新 html 文件）

  5. 将更新的文件提交到 GitHub 仓库上，重新访问博客页面 （https:{YOUR_NAME}.github.io）

附：一些细节

关于图片

在项目目录下创建 /pictures 目录，用于存放所有图片，当图片上传至 GitHub 仓库之后可以查看该图片的 URL，之后再写到文档模板中，可参考示例模板中关于插入图片的语句。

关于菜单

  • 创建空的菜单文件 MENU ，编辑内容如下

menu 1
  home      [index.html]
  about me  [me.html]
menu 2
  link      [http://jemdoc.jaboc.net/]

  • 在需要关联菜单的 .jemdoc 模板文件上，首行使用语句 jemdoc: menu{MENU} {CURRENT_FILE_NAME.html} ，表示在所生成的 html文件中附上菜单
